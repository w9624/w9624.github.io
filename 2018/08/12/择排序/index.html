选择排序：简单选择排序

  1. 定义：排序序列的记录个数为n，i取0,2,…,n-2,从所有后n-i-1个记录中找出排序码最小的记录，与第i个记录交换，执行n-1趟 后就完成了记录序列的排序的过程
  2. 算法思想：简单的说，每次选个最小值放在未排序部分的前面，n-1次之后排好序
    1. 对于n的记录，此时的i=0是第一个记录的下标，假设最小值在k=i位置，遍历后n-i-1个元素和k位置元素比较将较小的值得下标记为k，可以找到n个元素最小值的下标k，交换位置i和k的值，将最小值放到i位置，并且i++
    2. 重复n-1次上述步骤，将第n-1小的值放到位置n-1处，的得到升序序列
  3. 算法步骤：给一个数字序列升序排列：48，33，32，55，84，9，33，40
    1. 此时的i=0是第一个记录的下标，遍历后n-i-1=7个元素找到8个元素最小值的下标k
      48(k)，33，32，55，84，9，33，40 //k=i=0
      48，33(k)，32，55，84，9，33，40 //48 > 33, k = 1
      48，33，32(k)，55，84，9，33，40 //33 > 32, k = 2
      48，33，32(k)，55，84，9，33，40 //32 < 55, k = 2
      48，33，32(k)，55，84，9，33，40 //32 < 84, k = 2
      48，33，32，55，84，9(k)，33，40 //32 > 9, k = 5
      ·······
      48，33，32，55，84，9(k)，33，40 // k = 5
    2. 交换位置i和k的值，将最小值放到i位置，并且i++，重复步骤1
      9，33(k)，32，55，84，48，33，40 // k = i = 1
      重复步骤1可得：9，32(k)，33，55，84，48，33，40 // k = i = 1
    3. 重复n-3次步骤2，可得到升序
      9, 32, 33, 33, 40, 48, 55, 84

      /**
* 头尾同时进行的选择排序
* @param arr
*/
public static void select_one(int[] arr) {

int len  = arr.length;

for (int i = 0; i < arr.length / 2; i++) {
 //初始化最大最小值下标
 int min = i, max = i;
 //选出最小值和最大值
 for (int j = i + 1; j < arr.length - i; j++) {
     if (arr[min] > arr[j]) {
         min = j;
     }
     if (arr[max] < arr[j]) {
         max = j;
     }
 }
 //交换数组的位置,交换的时候一定要注意位置问题，很多人因为没有区分位置所以出错
 if (arr[max] == arr[min]) {  //数组已经有序
     break;
 }else {
     if (min == i && max == len - i - 1) {  //最大最小值位置正确
         continue;
     }else if (min == len - i - 1 && max == i) { //交换最大和最小值的位置刚好对调
         Tools.swap(arr, min, max);
     }else if (max == i) { //最大值在最小值要移动到的位置
         Tools.swap(arr, len - i - 1, max);
         Tools.swap(arr, i, min);
     }else {  //最小值在最大值要移动到的位置和其他情况
         Tools.swap(arr, i, min);
         Tools.swap(arr, len - i - 1, max);
     }
 }
}
}
  5. 算法分析（基础的选择排序）
    1. 稳定性：不稳定， 有算法实现可知，选择最小的数字，然后位置对调，例如 ：33， 33，22
      第一次排序：22, 33, 33
      第二次排序：22，33，33
      两个33的前后顺序发生改变，一种简单的情况就是，当在数字序列中有相同的两个数字，并且第2个数字后面位置有比第一个小的数字，两个数字顺序改变
    2. 时间复杂度：循环的次数为，数字移动的次数为 3n(n-1)/2，时间复杂度为 O[n^2]
    3. 空间复杂度：使用的辅助空间只有临时变量 temp, 空间复杂度为 O[1]
