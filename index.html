选择排序

发表于 2018-08-12 | 分类于 Algorithm ， sort

选择排序：简单选择排序

  1. 定义：排序序列的记录个数为n，i取0,2,…,n-2,从所有后n-i-1个记录中找出排序码最小的记录，与第i个记录交换，执行n-1趟 后就完成了记录序列的排序的过程
  2. 算法思想：简单的说，每次选个最小值放在未排序部分的前面，n-1次之后排好序
    1. 对于n的记录，此时的i=0是第一个记录的下标，假设最小值在k=i位置，遍历后n-i-1个元素和k位置元素比较将较小的值得下标记为k，可以找到n个元素最小值的下标k，交换位置i和k的值，将最小值放到i位置，并且i++
    2. 重复n-1次上述步骤，将第n-1小的值放到位置n-1处，的得到升序序列
  3. 算法步骤：给一个数字序列升序排列：48，33，32，55，84，9，33，40
    1. 此时的i=0是第一个记录的下标，遍历后n-i-1=7个元素找到8个元素最小值的下标k
      48(k)，33，32，55，84，9，33，40 //k=i=0
      48，33(k)，32，55，84，9，33，40 //48 > 33, k = 1
      48，33，32(k)，55，84，9，33，40 //33 > 32, k = 2
      48，33，32(k)，55，84，9，33，40 //32 < 55, k = 2
      48，33，32(k)，55，84，9，33，40 //32 < 84, k = 2
      48，33，32，55，84，9(k)，33，40 //32 > 9, k = 5
      ·······
      48，33，32，55，84，9(k)，33，40 // k = 5
    2. 交换位置i和k的值，将最小值放到i位置，并且i++，重复步骤1
      9，33(k)，32，55，84，48，33，40 // k = i = 1
      重复步骤1可得：9，32(k)，33，55，84，48，33，40 // k = i = 1
    3. 重复n-3次步骤2，可得到升序
      9, 32, 33, 33, 40, 48, 55, 84

    2. 两端同同时开始的选择排序

      /**
* 头尾同时进行的选择排序
* @param arr
*/
public static void select_one(int[] arr) {

int len  = arr.length;

for (int i = 0; i < arr.length / 2; i++) {
 //初始化最大最小值下标
 int min = i, max = i;
 //选出最小值和最大值
 for (int j = i + 1; j < arr.length - i; j++) {
     if (arr[min] > arr[j]) {
         min = j;
     }
     if (arr[max] < arr[j]) {
         max = j;
     }
 }
 //交换数组的位置,交换的时候一定要注意位置问题，很多人因为没有区分位置所以出错
 if (arr[max] == arr[min]) {  //数组已经有序
     break;
 }else {
     if (min == i && max == len - i - 1) {  //最大最小值位置正确
         continue;
     }else if (min == len - i - 1 && max == i) { //交换最大和最小值的位置刚好对调
         Tools.swap(arr, min, max);
     }else if (max == i) { //最大值在最小值要移动到的位置
         Tools.swap(arr, len - i - 1, max);
         Tools.swap(arr, i, min);
     }else {  //最小值在最大值要移动到的位置和其他情况
         Tools.swap(arr, i, min);
         Tools.swap(arr, len - i - 1, max);
     }
 }
}
}
  5. 算法分析（基础的选择排序）
    1. 稳定性：不稳定， 有算法实现可知，选择最小的数字，然后位置对调，例如 ：33， 33，22
      第一次排序：22, 33, 33
      第二次排序：22，33，33
      两个33的前后顺序发生改变，一种简单的情况就是，当在数字序列中有相同的两个数字，并且第2个数字后面位置有比第一个小的数字，两个数字顺序改变
    2. 时间复杂度：循环的次数为，数字移动的次数为 3n(n-1)/2，时间复杂度为 O[n^2]
    3. 空间复杂度：使用的辅助空间只有临时变量 temp, 空间复杂度为 O[1]

快速排序

发表于 2018-08-12 | 分类于 Algorithm ， sort

交换排序：快速排序

  1. 定义： 通过一趟排序将要排序的数据分割成独立的两部分，其中一部分的所有数据都比另外一部分的所有数据都要小，然后再按此方法对这两部分数据分别进行快速排序，整个排序过程可以递归进行，以此达到整个数据变成有序序列的过程

  2. 算法思想
    1.选择基准（哨兵）：在待排序列中，按照某种方式挑出一个元素，作为 “基准”（pivot）
    2.分割操作：以该基准在序列中的实际位置，把序列分成两个子序列。此时，在基准左边的元素都比该基准小，在基准右边的元素都比基准大
    3.递归地对两个序列进行快速排序，直到序列为空或者只有一个元素

  3. 排序步骤：给数字序列升序排列：44，23，34，33，22，23，56，54，66(对第二的23进行标注，用于判断稳定性)

    1. 首先选取数字序列的第一个数44为哨兵单独存储
      44     _（low），23，34，45，22，23，56，54，66（high） //4所在的位置现在为空，指针指向66
    2. 指针指向high，指针所在位置与哨兵44比较，偌大于哨兵则指针向前移动high–，直至小于等于哨兵，交换low和high位置数字
      44     _（low），23，34，45，22，23，56，54，66（high) //66 > 44 high–
      44     _（low），23，34，45，22，23，56，54（high），66 //54 > 44 high–
      44     _（low），23，34，45，22，23，56（high），54，66 //56 > 44 high–
      44     _（low），23，34，45，22，23（high），56，54，66 //23 < 44 交换，指针指向low，并low++
      44     23，23（low），45，33，22，_（high），56，54，66
    3. 指针指向low，指针所在位置与哨兵44比较，偌小于哨兵则指针向后移动low++，直至大于等于哨兵，交换low和high位置数字
      44     23，23，34（low），45，22，_（high），56，54，66 //23 < 44 low++
      44     23，23，34，45（low），22，_（high），56，54，66 //34 < 44 low++
      44     23，23，34，_（low），22（high），45，56，54，66 //45 > 44 交换，指针指向high，并high–
    4. 重复上述步骤直至low>=high，可得到以哨兵为分界的数组序列，此时low=high=哨兵位置
      44     23，23，34，22，44（low=high），45，56，54，66
    5. 此时以哨兵为界限，将数字序列分为两部分（哨兵除外），递归的对两个序列进行以上4个步骤获得有序数列
      22, 23, 23, 33, 34, 44, 54, 56, 66

  4. 算法实现

    1. 基本快速排序

    1. 优化基准（哨兵）的选取

      1. 随机数选取：使用随机下标的数字作为基准

      2. 三元法选取：选取开头，中间，末尾的三个元素的中间值作为基准

  5. 优化排序过程

    1. 在low-high的区间达到特定值的时候，使用选择排序
    2. 在分割结束后，将与基准相等的数字聚集，不再对这些元素进行分割
    3. 优化尾递归
    4. 并发

  6. 算法分析（基本快速排序）
    1.稳定性：不稳定，2,3在排序结果中顺序发生变化
    2.时间复杂度：O[nlog2 n]
    3.空间复杂度：O[log2 n]，递归树的高度

交换排序：通过以系列交换逆序元素进行排序的方法，包括冒泡排序和快速排序

冒泡排序
  1. 定义：冒泡排序是通过对 相邻元素 进行交换，逐步将待排序序列变成有序序列的过程
  2. 排序步骤：给一个数字序列升序排列：48，33，32，55，84，9，33，40
    1. 在第一趟冒泡排序中，从第一个记录开始，扫描整个序列，相邻数字为为降序则互换位置
      48，33，32，55，84，9，33，40 //48>33，交换
      33，48，32，55，84，9，33，40 //48>32，交换
      33，32，48，55，84，9，33，40 //48<55, 不交换
      33，32，48，55，84，9，33，40 ······
      33，32，48，55，84，9，33，40
      33，32，48，55，9，84，33，40
      33，32，48，55，9，33，84，40
      33，32，48，55，9，33，40，84
      在第一次冒泡结束之后，将最大值84放置在末尾
    2. 第二次冒泡排序中，对前7数字个进行同样操作
      33，32，48，9，33，40，55，84
      在第二次冒泡结束之后，将第二大数55放置在倒数第二个位置
    3. 第三次序中，对前6字个进行同样操作
      33，32，48，9，33，40，55，84
      在第三结束之后，将第三大的数48放置在倒数第三个位置
    4. 由此可以得出，反复进行7次之后得到升序序列
      9, 32, 33, 33, 40, 48, 55, 84

  3. 算法实现

    1. 简单冒泡排序

    2. 带标志位的冒泡排序

    3. 鸡尾酒排序（两端同时冒泡）

  4. 算法分析（只分析简单冒泡）

    1. 稳定性：稳定，从前向后比较，因为符合 arr[j] > arr[j + 1]进行交换，所以大小相同的数字不会交换前后位置

排序算法

发表于 2018-06-07 | 分类于 Algorithm ， sort
排序

  1. 定义： 排序是计算机内经常进行的一种操作，其目的是将一组“无序”的记录序列调整为“有序”的记录序列

  2. 分类： 根据排序时候所占存储器不同，可分为内部排序和外部排序

    • 内部排序：整个排序完全在内存中进行
    • 外部排序：由于数量过大，内存无法容纳全部数据，排序需要借助外部存储设备完成
排序算法的评估

  1. 稳定性： 待排序的序列中有两个或两个以上相同的关键字，在排序前和排序后，关键字顺序未发生变化则排序方法是稳定的，反之则排序方法是不稳定的

    • 证明一个排序方法是稳定的，要通过算法本身的步骤进行证明；证明排序方法是不稳定的，只需举出一个反例即可
    • 例子：待排序序列{R1, R2,… , Ri, …, Rj, …, Rk …, Rn}中存在三个关键字Ri=Rj=Rk（下标i<j<k），排序之后若Ri, Rj, Rk前后顺序未发生改变则使用的排序方法是稳定的，反之则使用的排序方法是不稳定的

  2. 时间复杂度： 简单的说就是描述该算法的运行时间，肯定不可能每次去跑一遍程序比较，于是就有了用语句频度来估算的方式

    1. 第一步我们需要知道什么是语句频度T(n) ：程序运行过程中语句执行的次数

      在上面的例子中,语句频度 T(n) = T1(n) + T2(n) = 2 n^2 + n*

    2. 第二步计算辅助函数 f(n)：使得当n趋近于无穷大时，T(n)/f(n)的极限值为不等于零的常数，则称f(n)是T(n)的同数量级函数，记为 T(n) = O(f(n))

      一般去除 T(n) = 2 n^2 + n 中的 低阶项和 首项系数可得到 f(n) = n^2*

    3. 最后 O(f(n)) = O(n^2) 获得渐进时间复杂度, 简称时间复杂度

    4. 常用的时间复杂度有：O(1) < O(log n) < 0(n) < O(n log n) < O(n^2) < O(2^n) < O(n!)

  3. 空间复杂度： 是对一个算法在运行过程中临时占用(辅助)存储空间大小的量度，记做S(n)=O(f(n)),
    与问题规模无关
    空间复杂度的计算与时间复杂度相似，时间复杂度使用的是语句频度，空间复杂度使用的是辅助空间的大小

    例如交换排序中常使用一个变量temp作为中间变量，空间复杂度为S(1)

    一般而言，时间和空间复杂度是可以进行一定的相互转换，

  4. 递归
    时间复杂度：递归总次数 * 每次递归的次数
    空间复杂度：递归的深度 * 每次递归所需要的辅助空间
